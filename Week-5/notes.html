Week 5 Notes

Winter 2023

This week we learned about testing and debugging in Javascript. There are different tpyes of errors which will determine how we go about the process. Here are the types of errors you can encounter:

  1. Syntax Errors - errors that are detected by the compiler
  2. Runtime Errors - errors that cause a program to terminate abnormally
  3. Logic Errors - occer when a program does not perform the way it was intended to

It's important to be able to recognize what errors we have so that we can properly test and debug them.

Concole.log()

One debugging method is console.log(). Using this allows us to display a value in the console so we can see what value is being claculated and can see if it's our desired outcome.

a = 5;
b = 6;
c = a + b;
console.log(c);

By doing this we will see that c is equal to 11 and and know whether or not our program is working correctly.

Breakpoints

Another method is by setting breakpoints. We can set breakpoints and when our code runs into one, it will stop executing and we can see the current values. We can then resume the program.

Debugger Keyword

The debugger keyword is similar to settting a breakpoint. It stops the execution of the program where it is placed before continuing.

var x = 15 * 5;
debugger;
document.getElementById("demo").innerHTML = x;
